# Table V2 问题清单

## 文档说明

本文档用于持续记录 `DataTable V2` 在设计、实现和业务接入过程中的实际问题，作为后续评审与迭代的输入。

主要目的：

- 沉淀可复现的问题现象，避免同类问题反复讨论却无结论。
- 区分“组件能力问题”和“页面使用方式问题”，帮助明确改进落点。
- 为后续方案讨论提供统一上下文，支持按优先级逐步改进。

记录约定：

- 每条问题尽量包含：现象、影响、当前处理、后续讨论点。
- 若已有临时修复，也要记录“是否只是页面级 workaround”。
- 问题状态建议使用：`待分析`、`待设计`、`进行中`、`已落地`、`已关闭`。

## 问题清单

### [DT-001] 工具栏同时暴露“重置筛选”和“重置状态”导致认知冲突

- 记录日期：2026-02-06
- 状态：已落地（待规范化）
- 类型：交互语义 / 最佳实践

现象：

- 页面工具栏中同时存在“重置筛选”和“重置状态”两个同级按钮。
- 两者视觉层级接近且文案相似，用户难以预判差异。

影响：

- 高频场景下更常用的是“重置筛选”，但用户会犹豫或误触“重置状态”。
- “重置状态”实际会影响更大范围（筛选、分页、排序、选择、列显隐、列宽、密度等），带来不符合预期的重置成本。

当前处理（已落地）：

- 页面工具栏收敛为两个图标按钮：`刷新` + `更多`（View Options）。
- `更多` 菜单统一承载：`密度切换`、`列设置`、`恢复表格默认设置`。
- 搜索框新增尾部清空按钮，支持一键清空搜索词。
- 每个筛选项新增独立清空按钮，支持逐项清除。

剩余待办（规范化）：

- 在设计文档与接入规范中明确：`resetAll` 为低频高影响动作，不建议主按钮常驻。
- 评估是否在 `DataTablePreset` 中内置该交互模式，减少业务侧重复接线。
- 统一 `resetAll` 是否需要确认交互（确认弹窗/二次确认文案）。
